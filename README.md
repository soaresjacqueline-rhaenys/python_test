# 🐍 python_test

Projeto de testes automatizados com Python desenvolvido durante o **Bootcamp de Analista de QA da TripleTen**, com foco na escrita e execução de testes unitários para validação de funcionalidades de uma aplicação.

---

## 🎯 Objetivo do Projeto

Desenvolver testes automatizados em Python para validar o comportamento de funções de uma aplicação, praticando a criação de casos de teste unitários, cobertura de cenários positivos e negativos, e organização de código de teste seguindo boas práticas de QA.

---

## ✅ Resultado

- Criação de testes unitários para validação de operações matemáticas (cálculos e multiplicações)
- Cobertura de cenários positivos e negativos nas funções testadas
- Aplicação dos conceitos de automação de testes com Python puro
- Estrutura de projeto simples e funcional para fins de aprendizado e prática

---

## 🛠️ Ferramentas Utilizadas

| Ferramenta | Finalidade |
|---|---|
| Python | Linguagem de desenvolvimento e testes |
| pytest | Framework de execução de testes |
| PyCharm | IDE de desenvolvimento |
| Git / GitHub | Controle de versão |

---

## 📁 Estrutura do Projeto

```
python_test/
│
├── main.py                # Arquivo principal da aplicação
├── application.py         # Lógica da aplicação a ser testada
├── calc_test.py           # Casos de teste para operações de cálculo
├── calc_multiply_test.py  # Casos de teste para operações de multiplicação
└── .idea/                 # Configurações do PyCharm
```

---

## 🧪 Casos de Teste

Os testes cobrem os seguintes cenários:

- ✔️ Validação de operações de **cálculo** com diferentes entradas
- ✔️ Validação de operações de **multiplicação** com diferentes entradas
- ✔️ Testes com **valores esperados** (cenários positivos)
- ✔️ Testes com **valores inesperados** (cenários negativos/limite)

---

## 📚 O Que Aprendi

- ✔️ Escrita de **testes unitários** com Python
- ✔️ Uso do **pytest** para execução e organização de testes
- ✔️ Como estruturar um projeto de testes separando **lógica da aplicação** e **casos de teste**
- ✔️ Cobertura de **cenários positivos e negativos** em testes automatizados
- ✔️ Primeiros passos na **automação de testes** com Python
- ✔️ Boas práticas de **organização e nomenclatura** de arquivos de teste

---

## 🚀 Como Executar

```bash
# Clone o repositório
git clone https://github.com/soaresjacqueline-rhaenys/python_test.git

# Acesse a pasta do projeto
cd python_test

# Execute os testes
pytest calc_test.py
pytest calc_multiply_test.py

# Ou execute todos de uma vez
pytest
```
